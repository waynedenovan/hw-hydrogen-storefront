## Resolution log (2607150700, claude)

VERSION bumped: 2026.07.9 → 2026.07.10

### Debug ritual followed
1. `search_errors` / MCP-Bridge `qdrant-find` first (query: "customer account login redirect_uri mismatch oauth authentication error hw-hydrogen-storefront") — found prior session 2606021433 which set up Customer Account API OAuth for **local dev only** (via ngrok tunnel `alfredia-prebudgetary-nonperceptibly.ngrok-free.dev`). No prior session had registered the **production** domain `https://hoseworld.store`.
2. `shopify-dev-mcp` (`learn_shopify_api` api=`hydrogen`, then `search_docs_chunks`) confirmed the standard pattern: `context.customerAccount.authorize()` is "the Remix loader configured as the redirect URI within the Customer Account API settings in admin" — i.e. the callback URL must be explicitly allow-listed in Shopify Admin, it is not inferred/auto-trusted from the request.
3. Reproduced live via Playwright MCP: navigated to `https://hoseworld.store/account` → app correctly redirects to
   `https://shopify.com/authentication/79356526823/oauth/authorize?client_id=ac69523f-e849-490a-b22f-49d7a6920a4a&...&redirect_uri=https%3A%2F%2Fhoseworld.store%2Faccount%2Fauthorize&...`
   → Shopify returns "Something went wrong — redirect_uri mismatch" (title "hw-storefront-ui", matching the client_id's shop). Screenshot matches `app/errors/260714220538-errors.png` exactly (same client_id/shop id, same error copy).

### Root cause
**Not a code bug.** `app/lib/context.ts` (`useCustomAuthDomain: true`) and `app/routes/account_.login.tsx` / `account_.authorize.tsx` all use the standard Hydrogen SDK pattern and correctly build `redirect_uri=https://hoseworld.store/account/authorize` from the live request origin. The failure is a **Shopify Admin configuration gap**: the Headless channel's Customer Account API "Application setup" only has the local-dev ngrok callback/origin registered (from session 2606021433) — the production callback URI, JavaScript origin, and logout URI for `https://hoseworld.store` were never added when the storefront went live on that domain.

Verified container is current (created 2026-07-14 21:59, error screenshot timestamped 2026-07-14 22:05 — after container start), so this isn't a stale-build issue either.

### Resolution — requires manual admin action (Wayne)
No code change was made or is needed. In Shopify Admin → Headless channel → Customer Account API → Application setup, add for the production app client (`client_id: ac69523f-e849-490a-b22f-49d7a6920a4a`, shop `79356526823`):
- Callback URI: `https://hoseworld.store/account/authorize`
- JavaScript origin: `https://hoseworld.store`
- Logout URI: `https://hoseworld.store`

Keep the existing ngrok dev-tunnel entries alongside these — Customer Account API supports multiple registered URIs per client, so prod and local dev can coexist.
