## 4. Naming convention: strip type codes, new {type} field

Supplier data pattern (confirmed against the local DB): Sub Collections are
"XX NAME" (2-letter code, e.g. "AC BINDING"), Sub-Cat Collections are
"XXYY NAME" (4-letter code whose first 2 letters are the parent sub's code,
e.g. "ACAE TWINE", "FZAA BAILING TWINE").

CRITICAL constraint discovered: cleaned names are NOT unique — "AC BINDING" and
"FB BINDING" both clean to "BINDING", and "FZ AGRICULTURAL" cleans to
"AGRICULTURAL" which collides with the fixed main Collection "Agricultural".
So the coded names REMAIN the identity everywhere (Shopify collection titles,
handles/URLs, SubCollectionMap keys, admin app lists) and the cleaned name is
display-only metadata. This is what keeps the existing integration unbroken.
