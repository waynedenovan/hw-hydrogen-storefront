There are two primary reasons a client drops a connection with error code 0 during a data import:

If your local application at http://localhost:3458 took longer than 100 seconds to process the import file, Cloudflare cut the client connection.

    Browser/Client Timeout: If a script or browser was handling the import, it may have had a built-in timeout limit (e.g., 30 or 60 seconds) and gave up before your server finished processing.

How to Fix It

Since processing large dataset imports usually takes time, you have a few ways to resolve this:
Fix A: Stream the Response (Recommended)

If your backend server (localhost:3458) accepts the file and processes it entirely before sending an HTTP response back, it will trigger timeouts.

    The Fix: Have your application respond immediately with an HTTP 202 Accepted status and process the file in the background (asynchronous processing), OR send chunked/streaming progress updates back to the browser so the connection stays active.

Fix B: Increase Cloudflare's Timeouts (Enterprise Only) - Not viable

If you are on a Cloudflare Enterprise plan, you can increase the Proxy Read Timeout zone up to 600 seconds via Page Rules or the API. Note: If you are on a Free, Pro, or Business plan, this limit is hard-capped at 100 seconds.

Fix C: Bypass Cloudflare for Large Imports - Not viable

If this is an internal administrative task, bypass the tunnel or Cloudflare proxy entirely for large file imports by hitting the local server IP/port directly (e.g., via a VPN), or configure a specific subdomain for imports that is set to DNS Only (Grey Clouded) in your Cloudflare dashboard.
