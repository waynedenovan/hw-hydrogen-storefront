## Step 2: Documentation and exit

### Root cause (confirmed via direct Storefront API queries, not guesswork)

Queried `hw-storefront-ui.myshopify.com`'s Storefront API directly for the `irrigation-pumps`
collection and for all 250 live products:
- `featuredImage`/`images` were empty for 100% of products.
- The `app`-namespaced metafields the storefront read (`brand`, `sub_collection`,
  `sub_cat_collection`, `supplier_name`, `external_product_id`) were **null for 100%
  of products**, even though `vendor` (e.g. "PASCALI") was set correctly.

Traced the write side in the sibling admin app
`hw-storefront-ui-node-docker/app/services/product.server.js` (`mapExternalToShopify`):
it wrote these metafields under Shopify's **app-reserved `$app` namespace**. Confirmed
against Shopify's own docs (via `shopify-dev-mcp`): by default the Storefront API can't
read *any* metafield — a metafield **definition** must exist with
`access.storefront: PUBLIC_READ`, and none existed anywhere in the repo
(`metafieldDefinitionCreate` appeared nowhere). Since this data needs to be read by a
*different* app (the Hydrogen storefront's Storefront API client) than the one that
wrote it, `$app` was also the wrong namespace choice — Shopify's docs specify a
merchant-owned custom namespace (`"custom"`) for exactly this cross-app case. This one
bug explained both the missing images (the local-disk image fallback depends on
`supplier_name`/`external_product_id`, which were null for the same reason) and the
"only Other" section grouping (`sub_collection`/`sub_cat_collection` were null).

### Fix applied

1. `hw-storefront-ui-node-docker/app/services/product.server.js`: all 12 metafield
   writes in `mapExternalToShopify` changed from `namespace: "$app"` to
   `namespace: "custom"` (via a `METAFIELD_NAMESPACE` constant). Added
   `ensureStorefrontMetafieldDefinitions(admin)`, which creates the 12 corresponding
   `PUBLIC_READ` Storefront metafield definitions (idempotent).
2. `hw-storefront-ui-node-docker/scripts/backfill-custom-namespace.mjs` (new): one-time
   script that ran the definitions setup, then re-synced metafields (via the existing
   `syncProductMetafields`, not a full product re-sync, to avoid touching
   title/price/status/inventory) for every locally-tracked product. Result: **86/86**
   locally-tracked products re-synced successfully; verified via Storefront API that
   `custom`-namespace metafields now resolve (e.g. Pascali products now return
   `sub_collection: "JA PUMPS"`, `sub_cat_collection: "JAAM SUBMERSIBLE PUMPS"`, exactly
   matching the example in Step 1 above).
3. `hw-hydrogen-storefront-node-docker`: `app/routes/($locale).collections.$handle.tsx`
   and `app/routes/($locale).products.$handle.tsx` reader queries changed from
   `namespace: "app"` to `namespace: "custom"` to match.
4. Collection page now groups Brand → Sub-collection → Sub-cat (three nested
   `CollapsibleSection` levels) instead of a flat two-level `subCollection >
   subCatCollection` tree with no brand grouping.
5. Filter UI: added a dual-handle price **slider** (kept the existing min/max number
   inputs alongside it) and a "Filters" icon/button that toggles the filter sidebar —
   the Brand/Sub Collection/Sub-Cat Collection checkbox filters already existed in code
   and just needed real (non-null) data to populate, which the namespace fix provides.
6. Styling: created `.claude/skills/collection-page-style/SKILL.md` documenting the
   Product-details page's dark-translucent-card tokens (`rgba(50,50,50,0.65)` wrapper,
   white/gray-300 text), and applied the same wrapper + text-color overrides to the
   Collection page so both pages now match.
7. `app/components/ProductCard.tsx`: added dev-only `console.warn` calls (via
   `useEffect`, not on every render) when neither the Shopify `featuredImage` nor the
   local `/media/suppliers/...` fallback resolves, so future "no image" cases are
   traceable instead of silently showing the placeholder.
