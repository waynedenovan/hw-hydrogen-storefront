## 1. Cloudflare tunnel suggestions file (2607170850) — verdict and fix

### Suggestion 1: "Slow 404 image requests (750ms+)" — AGREED, fixed, live-verified

**Actual root cause (sharper than the file's "booting the entire app"):** every
`/media/suppliers/*` request ran through the Express catch-all into the full
react-router SSR pipeline, and — the dominant cost — `server.ts` called
Hydrogen's `storefrontRedirect()` on **every 404**. Confirmed via shopify-dev-mcp
docs: storefrontRedirect "queries the Storefront API to see if there is any
redirect created for the current route". So each missing image = one live
Shopify API round-trip (~350–750ms, matching hw-hydrogen.log exactly: 404s
353–755ms vs 200s 2.3–8.6ms). Dozens of parallel missing-image lookups per
page is what triggered the Cloudflare "stream canceled by remote" errors.

**Fix (`server.ts`):**
1. `express.static('media/suppliers', {maxAge:'1y', immutable, index:false, redirect:false})`
   mounted at `/media/suppliers`, followed by an instant `404` handler —
   registered after morgan (so missing-image log lines are kept for tracking
   which images are still to be collected) but before the SSR catch-all.
2. Belt-and-braces: the catch-all now skips `storefrontRedirect()` for any
   `/media/` path (no Shopify redirect can exist for those).
3. `app/routes/media.suppliers.$.tsx` kept as a now-unreachable fallback.

**Constraints honored:** images stay OUTSIDE the container (host bind mount
`../hw-hydrogen-storefront/media:/app/media:ro`); express.static reads the
mount per-request, so images Wayne is still collecting can be dropped into the
host folder and serve immediately — verified live with a drop-in test file
(200 in 2.5ms, zero restart/rebuild).

**Live verification (after `docker compose build && up -d`):**
- miss: 404 in **0.4–1.1ms** (was 353–755ms — ~500x faster); hit: 200 in 2–7ms, correct Content-Type
- path traversal (`/media/suppliers/../../package.json`) → 404
- via https://hoseworld.store: hit 200 / miss 404, no stream errors
- a real visitor's miss (`MSSET120605AN_3.jpg`) already served at 0.36ms in the container log
