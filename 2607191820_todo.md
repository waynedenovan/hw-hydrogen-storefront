## Task B — header Search icon / server-side weighted search

**Root cause the icon "did not work"**: the icon itself was fine (opens the search aside via `useAside`). The aside's predictive machinery (`SearchFormPredictive` → GET `/search?q=&limit=5&predictive=true`) expected a `PredictiveSearchReturn` `{type, term, result:{total, items}}` payload, but the `/search` loader ignored `predictive`/`limit` entirely and returned `{searchTerm, products}` — and no predictive GraphQL query existed anywhere in the repo (`lib/search.ts` referenced never-generated `PredictiveSearchQuery`/`RegularSearchQuery` types). So `fetcher.data.result` was always undefined and the dropdown permanently rendered the empty state.

**Fix — new server-side weighted engine** (`app/lib/searchIndex.server.ts`, dynamic-imported in the loader per ERR-IMPORT-001):
1. Pages the whole catalog into an in-memory index via two Storefront API queries (both validated with shopify-dev-mcp against 2026-04): products (title, description, `custom.erp_long_description`, brand/sub_collection/sub_cat_collection/type/msq/supplier_name/external_product_id metafields, price, image, first variant) and collections (title, description, `custom.collection_role`/`display_name`/`parent_collection`). 5-minute module-level TTL cache; 250/page; capped.
2. Scores every entity per query token with explicit weights — product code 12 (compared with " " and "/" excluded, same normalization as Task A, exact-match doubled), title 10 (+15 phrase-in-title, +5 starts-with), brand 8, sub/sub-cat/type-code 6, productType/vendor 4, description/erp long description 2; whole-word match earns full weight, substring 60%. Results sorted best-match-first. Collections scored on title/display_name/description; brands aggregated from product metafields with product counts.
3. Predictive thumbnails resolved server-side (Shopify featuredImage, else fs-verified local supplier file) so the dropdown never renders dead images (same pattern as the 2607161525 gallery fix).

**Wiring**: `($locale).search.tsx` loader now answers both modes — `?predictive=true&limit=N` returns the aside's dropdown shape (products, collections, query suggestions for the datalist; articles/pages empty), plain `?q=` returns the full weighted page. `lib/search.ts` rewritten with concrete types (removing the dead generated-type imports). Full page renders grouped sections — Collections (role=main), Sub Collections (everything else incl. sub-cat, showing cleaned display names), Brands (chips with product counts linking to `/search?q=<brand>`), Products (top 48 as ProductCard grid) — inside `.page-card page-card--wide` per the collection-page-style skill; new CSS in app.css (`.search-page-form`, `.search-section`, `.search-brand-*`).

**Tests**: new `app/lib/__tests__/searchIndex.server.test.ts` 10/10 (weighting order, brand/sub-cat matching, code search with "/" and " ", predictive/regular shapes).
