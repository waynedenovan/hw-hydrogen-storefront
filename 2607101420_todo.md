This session is a third occurrence of the stale-container pattern.

### Root cause
**Not a code bug.** The fix described in `app/errors/sugested_solution.md` (fire-and-forget POST + background batch processing) was already implemented and committed to `hw-storefront-ui-node-docker` today, commit `00ce858` (2026-07-10 10:59:24 +02:00) — see `app/services/import-run.server.js` (`startImportRun`/`launchRun`/`executeRun`, persisted `ImportRun`/`ImportBatchFailure` Prisma models with `offset`/`phase`/`heartbeatAt` for resume), and `app/routes/app.import.jsx` action (`start-supplier-import` intent returns immediately; `import-run-status` intent polls every 3s). The `/app/import.data → 63.8s` blocking POSTs, upstream TCP retransmit bursts to `23.227.38.74:443` in the tcpdump captures, and the "no response available: GraphQL Client: fetch failed" error are all consistent with the *old*, pre-refactor synchronous code — because that's what was still running.

Verified via `docker inspect`:
- Container `hw-storefront-ui-node-docker-storefront-ui-1` was created **2026-07-10 07:27:16 UTC**.
- Fix commit `00ce858` landed **2026-07-10 08:59:24 UTC** (10:59:24 +02:00) — **3.5 hours after** the container was created.
- The backing image had last been rebuilt **2026-07-08 16:42:47 UTC**, two days earlier.

So the container was serving code from before the 2607081815 session even, let alone today's fix. `docker compose build` alone does nothing to a running container — it must also be recreated (`docker compose up -d`). This is the exact `stale-container-runs-pre-fix-code` pattern from the 2607081815 incident, recurring because the rebuild+recreate step was never run after commit `00ce858`.

Wayne's read of the screenshots ("total stayed at 13379, should have dropped to 13021") was a red herring — `13379` is the total product count in the import file, not remaining/progress; it doesn't decrease. The real symptom is that the **progress numerator** reset from 359 to 15 on restart instead of resuming — which is exactly what you'd expect from the old code, which had no persisted resumable run state (that's what commit `00ce858` added).
