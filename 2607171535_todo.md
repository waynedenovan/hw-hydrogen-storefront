### 1. Product Cards — MOQ ribbon alignment — FIXED, verified live
Root cause: `.moq-ribbon` (app.css) was positioned with fixed pixel offsets
(`top: 12px; left: -38px`) before its `rotate(-45deg)`. The strip's unrotated
centre sat at (42px, ~25px) — farther from the left edge than from the top — so
after rotation the wording sat closer to the image's top edge ("y" in Qty
touching it).
Fix (app.css): position the strip at the corner and place it along the corner
diagonal instead: `top:0; left:0; transform: translate(-50%, -50%)
rotate(-45deg) translateY(44px)`. Because the offset is along the diagonal
itself, the text is equidistant from the top and left edges BY CONSTRUCTION,
regardless of the strip's content height.
Verified via Playwright getBoundingClientRect: label centre now exactly 25.9px
from top AND 25.9px from left; element screenshot confirms both rows fully
inside the band.

### 4. Tunnel errors + log review — DIAGNOSED; one real bug found & fixed
- 260717153500-cloadflare.md (the 2 tunnel ERR lines): ONE request to
  /app/import.data was "canceled by remote with error code 0" — the CLIENT
  (browser) closed the stream, most likely a page close/refresh mid-poll. The
  md file's 100s-Cloudflare-timeout theory does not fit the data:
  hw-storefront.log shows every import.data request completing in ~450-510ms
  (the server-side job-runner batching already keeps requests far under the
  100s edge ceiling) and the import continuing normally after the two ERRs.
  hw-storefront-ui: working normally, no action needed.
- hw-hydrogen.log: the /media/suppliers/agri 404 lines are the intended
  instant-404 logging (session 2607170910). BUT the intermittent 500s on
  product/collection pages were a REAL recurring bug: every single 500
  completed in a 255.3–255.8ms band — the signature of Node 20+/22's
  happy-eyeballs CONNECT timeout (autoSelectFamilyAttemptTimeout, default
  250ms). Any TCP connect to Shopify whose SYN-ACK took >250ms was aborted,
  all fallback attempts failed inside the same budget, and the loader 500'd
  with "TypeError: fetch failed / AggregateError [ETIMEDOUT]" (66 occurrences
  in 2 days of container logs; the existing setDefaultResultOrder('ipv4first')
  only fixed DNS ordering, not the connect-side race).
  FIX (server.ts): setDefaultAutoSelectFamily(false) +
  setDefaultAutoSelectFamilyAttemptTimeout(2000). Deployed with this build.
