### Root causes found (two independent bugs, both verified live before fixing)

**Bug 1 — large screen: text stuck in a narrow column (errors 260716161934 / 260716165322)**
- `app/styles/app.css` `.product { display: grid; grid-template-columns: 1fr 1fr }` (>=45em) is leftover skeleton CSS from when `.product-image`/`.product-main` were direct children of `.product`. They now live inside the single `.product-layout` wrapper, so the two-column split parked `.product-layout` in the LEFT column only, and `.product-layout` then split that half again 2fr/3fr.
- Measured live (Playwright, 1600px viewport): `.product-layout` = 547px inside a 1190px-wide `.product`; the text column (`.product-main`) got 309px of a 1600px screen (~19%).

**Bug 2 — small screen: content extending past the card's and screen's right edge (errors 260716161757 / 260716161814)**
- `.product-dimensions-table table { min-width: 260px }` propagated up through grid items that all had the default `min-width: auto` (`.product-main` → `.product-layout` → `.product`), so the whole column could never shrink below ~260px + gutters.
- At desktop widths this stayed hidden (track was 309px, just above 260). On narrow *effective* viewports it overflowed: reproduced live at a 280px viewport — `document.scrollWidth` 313 vs 280, whole `.product-layout` escaping the card. Wayne's phone hits this at normal sizes because browser zoom / OS text scaling shrinks the effective CSS viewport and scales the em-based breakpoints (his landscape screenshot shows the <45em stacked layout at ~800px physical width — the tell).
- Made worse by `html { overflow-x: clip }`: the overflowed content is unreachable (no horizontal scroll), so it just looks cut off.

**Gap 3 — the picture carousel in the spec was never built**: gallery thumbnails rendered but were not clickable/expandable.

### Fixes applied

1. `app/styles/app.css` — `.product` is now a plain block (removed the leftover `1fr 1fr` grid) with `overflow-x: clip` as a last-resort in-card clip guard, plus `padding-inline: 3%` at >=45em to enforce the 90%-width / 5%-inset spec.
2. `app/styles/app.css` — `.product-layout` tracks changed from `1fr` / `2fr 3fr` to `minmax(0, 1fr)` / `minmax(0, 2fr) minmax(0, 3fr)`; `.product-image`/`.product-main` given `min-width: 0` and `overflow-wrap: break-word`. The dimensions table now shrinks with the column and scrolls horizontally inside its own wrapper as originally designed.
3. `app/routes/($locale).products.$handle.tsx` — removed the `max-w-7xl` cap on `.product` (text must keep expanding with the screen; the 90%/5% rule is enforced in CSS instead).
4. `app/routes/($locale).products.$handle.tsx` — added the spec'd picture carousel: gallery images (main supplier image + `{productId}_1`-style extras) are now buttons; clicking opens a full-screen overlay (`GalleryCarousel`) with the image at max 90vw/85vh, prev/next (wrapping) when more than one image exists, close button, backdrop-click close, and Escape/Arrow key support. Closing returns thumbnails to their minimised grid.
