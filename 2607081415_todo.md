### Product Detail Page
- **Same formatting, no filter icon** (`260708130033-errors.jpeg`): PDP already had no filter
  icon (never did) — confirmed, no change needed there.
- **New responsive-gutter standard**: added shared `.page-card` / `.page-card--narrow` (products,
  80% desktop) / `.page-card--wide` (collections, 90%) classes in `app/styles/app.css`, replacing
  both routes' inline-style outer wrapper. Gutter/padding now shrink in steps as the viewport
  narrows instead of staying a fixed 80/90%, matching the mobile reference screenshot. Documented
  in `.claude/skills/collection-page-style/SKILL.md` as the project-wide standard for any new
  `.page-card`-based page.
- **{desc} gets 60% of the central area** (`260708130114-errors.png`): new `.product-layout` CSS
  grid (`2fr 3fr` at ≥45em, single column below) replaces the old even 50/50 Tailwind grid.
- **Product Details block**: added below the existing description — "Part No.: {cleansku}" line,
  then a deduplicated concatenation of shortDescription / b2cShortDescription / b2cDescription /
  erpShortDescription / erpLongDescription, each field separated by a divider line, empty fields
  (or fields left with no unique sentences after dedup) dropped entirely. **Issue found & fixed
  during live testing**: the naive implementation deduped only *among* the 6 combined fields, so
  for any product with data in only `descriptionHtml`/longDescription (the common case before the
  backfill below), the new block exactly re-printed the paragraph already shown as {desc} above
  it. Fixed by seeding the dedup set with {desc}'s own sentences first, so a field identical to
  what's already visible collapses to nothing instead of duplicating it. Logged in
  `project_errors.db` as `combined-desc-block-dupes-visible-desc-without-seeded-dedup`.
- **Dimensions table**: replaced the old `<ul>` "Specifications" list (labeled "cm") with a
  responsive `<table>` (`overflow-x-auto` wrapper) — Weight (kg), Length/Width/Height (mm).
  Checked raw supplier JSON (e.g. length "1005", width/height "35" for a real product) to confirm
  the source data is genuinely millimetres/kilograms, so relabeling from "cm" was a correctness
  fix, not just following the spec literally.
- **missing_images.md**: new `scripts/generate-missing-images-report.mjs` queries the Storefront
  API for `supplier_name: "agrinet"` products, checks `featuredImage` + local
  `media/suppliers/agri/{id}.jpg` (that path is a symlink to
  `../hw-hydrogen-storefront/media/suppliers`, ~41k real files — it looked empty from one check
  method but resolves fine via Node's `fs`), and regenerates
  `suppliers/agrinet/prod/missing_images.md` (flat `- {productId}` list) every run. First run: 41
  Agrinet products checked, 14 missing an image.
