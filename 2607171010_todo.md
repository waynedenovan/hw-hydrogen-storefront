## Resolution (VERSION 2026.07.18)

### Plan
This project's established convention (sessions 2607161525/2607161650/2607170910, per MCP-Bridge memory) treats "Npt" as a literal ±N px adjustment to `font-size`, applied per responsive tier, as plain unlayered CSS in `app.css` — Tailwind utility classes on h1/h2/h3/p are silent no-ops here because `@import "tailwindcss"` puts them in CSS cascade layers while `reset.css`'s bare element rules are unlayered and always win.

Scope decision: "Detailed Product page" means `.product-main` only. The existing font rules were shared verbatim with `.product-card` (collection-page cards) for `p`/`.product-description`/`.product-price-display`, so those shared selectors were split into `.product-card …` (left unchanged) and `.product-main …` (bumped +2px) pairs at all three tiers, to avoid also resizing the Product Card.

"Product detail information" was interpreted as the body/paragraph and table content under `.product-main`: the description (`.product-description`), the "Product Details" block (Part No. + deduped detail sentences, both `<p>` tags), and the Specifications table (`table`). Headings (`h1` title, `h3` "Product Details"/"Specifications" subheads) were left unchanged — the instruction named "information" and "price" specifically, and the h3 rule is shared with `.product-card h3` (title-equivalent), out of scope for a price/info-only change. "Price text" = `.product-price-display`, shared between `ProductCard.tsx` and the detail page's price div; same split-selector treatment.
